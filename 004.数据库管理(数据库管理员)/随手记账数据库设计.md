## 2. 表设计

### 2.1 用户表 (users)

存储用户注册信息。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名（至少2字符） |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希（bcrypt） |
| nickname | VARCHAR(50) | NULL | 昵称 |
| avatar_url | VARCHAR(500) | NULL | 头像 URL |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| last_login_at | DATETIME | NULL | 最后登录时间 |
| is_active | TINYINT(1) | NOT NULL, DEFAULT 1 | 是否启用（0=禁用，1=启用） |

**索引：**
- `idx_username` ON (username)
- `idx_created_at` ON (created_at)

---

### 2.2 账单表 (transactions)

记录所有收支及转账流水。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| user_id | VARCHAR(36) | NOT NULL, FK | 用户 ID |
| type | ENUM('expense','income','transfer') | NOT NULL | 交易类型：支出/收入/转账 |
| amount | DECIMAL(12,2) | NOT NULL | 金额（正数） |
| category_id | VARCHAR(36) | NULL, FK | 分类 ID（转账时可为NULL） |
| account_id | VARCHAR(36) | NOT NULL, FK | 账户 ID（支出来源/收入去向） |
| to_account_id | VARCHAR(36) | NULL, FK | 目标账户 ID（仅转账时使用） |
| transaction_date | DATE | NOT NULL | 交易日期 |
| transaction_time | TIME | NULL | 交易时间 |
| note | VARCHAR(500) | NULL | 备注 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| is_deleted | TINYINT(1) | NOT NULL, DEFAULT 0 | 软删除标记（0=正常，1=已删除） |

**索引：**
- `idx_user_id` ON (user_id)
- `idx_user_date` ON (user_id, transaction_date)
- `idx_user_category` ON (user_id, category_id)
- `idx_type` ON (type)
- `idx_is_deleted` ON (is_deleted)

**约束：**
- CHECK (amount > 0)

---

### 2.3 分类表 (categories)

管理收支分类，支持父子分类。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| user_id | VARCHAR(36) | NULL, FK | 用户 ID（NULL 表示系统默认分类） |
| type | ENUM('expense','income') | NOT NULL | 分类类型：支出/收入 |
| name | VARCHAR(50) | NOT NULL | 分类名称 |
| icon | VARCHAR(100) | NOT NULL | Material Symbols 图标名 |
| color | VARCHAR(7) | NULL | 颜色 HEX 值（如 #FF5722） |
| parent_id | VARCHAR(36) | NULL, FK | 父分类 ID（NULL 表示一级分类） |
| sort_order | INT | NOT NULL, DEFAULT 0 | 排序序号（越小越靠前） |
| is_system | TINYINT(1) | NOT NULL, DEFAULT 0 | 是否系统默认（0=用户自定义，1=系统默认） |
| is_hidden | TINYINT(1) | NOT NULL, DEFAULT 0 | 是否隐藏（0=显示，1=隐藏） |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `idx_user_type` ON (user_id, type)
- `idx_parent` ON (parent_id)
- `idx_sort_order` ON (sort_order)

---

### 2.4 账户表 (accounts)

管理用户的资金账户。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| user_id | VARCHAR(36) | NOT NULL, FK | 用户 ID |
| name | VARCHAR(50) | NOT NULL | 账户名称 |
| type | ENUM('cash','electronic','bank','credit','other') | NOT NULL | 账户类型 |
| icon | VARCHAR(100) | NULL | 图标名 |
| initial_balance | DECIMAL(12,2) | NOT NULL, DEFAULT 0 | 初始余额 |
| current_balance | DECIMAL(12,2) | NOT NULL, DEFAULT 0 | 当前余额 |
| is_default | TINYINT(1) | NOT NULL, DEFAULT 0 | 是否默认账户（0=否，1=是） |
| is_active | TINYINT(1) | NOT NULL, DEFAULT 1 | 是否启用（0=禁用，1=启用） |
| sort_order | INT | NOT NULL, DEFAULT 0 | 排序序号 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `idx_user_id` ON (user_id)
- `idx_is_default` ON (is_default)
- `idx_sort_order` ON (sort_order)

**账户类型说明：**
- `cash`: 现金
- `electronic`: 电子钱包（微信/支付宝）
- `bank`: 银行卡
- `credit`: 信用卡
- `other`: 其他

---

### 2.5 预算表 (budgets)

管理用户的月度预算。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| user_id | VARCHAR(36) | NOT NULL, FK | 用户 ID |
| year_month | VARCHAR(7) | NOT NULL | 预算月份（格式：YYYY-MM） |
| category_id | VARCHAR(36) | NULL, FK | 分类 ID（NULL 表示总预算） |
| budget_amount | DECIMAL(12,2) | NOT NULL | 预算金额 |
| alert_threshold | DECIMAL(5,2) | NOT NULL, DEFAULT 80.00 | 预警阈值（百分比，如 80 表示 80%） |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `idx_user_month` ON (user_id, year_month)
- `idx_user_category_month` ON (user_id, category_id, year_month)

**约束：**
- UNIQUE (user_id, category_id, year_month)

---

### 2.6 提醒表 (reminders)

管理用户的记账提醒设置。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| user_id | VARCHAR(36) | NOT NULL, FK | 用户 ID |
| reminder_time | TIME | NOT NULL | 每日提醒时间 |
| reminder_message | VARCHAR(200) | NULL | 提醒文案 |
| is_enabled | TINYINT(1) | NOT NULL, DEFAULT 1 | 是否启用（0=关闭，1=开启） |
| exclude_weekends | TINYINT(1) | NOT NULL, DEFAULT 0 | 周末不提醒（0=提醒，1=不提醒） |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

**索引：**
- `idx_user_id` ON (user_id)

---

### 2.7 备份记录表 (backups)

记录用户数据备份历史。

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID 唯一标识 |
| user_id | VARCHAR(36) | NOT NULL, FK | 用户 ID |
| backup_type | ENUM('manual','auto') | NOT NULL | 备份类型：手动/自动 |
| file_url | VARCHAR(500) | NULL | 备份文件存储路径 |
| file_size | BIGINT | NULL | 文件大小（字节） |
| status | ENUM('success','failed') | NOT NULL | 备份状态 |
| error_message | VARCHAR(500) | NULL | 错误信息（失败时） |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 备份时间 |

**索引：**
- `idx_user_id` ON (user_id)
- `idx_created_at` ON (created_at)

## 3. SQL 脚本

### 3.1 创建数据库

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS suishouji
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

USE suishouji;
```

### 3.2 创建用户表

```sql
-- 用户表
CREATE TABLE IF NOT EXISTS users (
  id VARCHAR(36) PRIMARY KEY,
  username VARCHAR(50) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  nickname VARCHAR(50) NULL,
  avatar_url VARCHAR(500) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  last_login_at DATETIME NULL,
  is_active TINYINT(1) NOT NULL DEFAULT 1,
  INDEX idx_username (username),
  INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.3 创建分类表

```sql
-- 分类表
CREATE TABLE IF NOT EXISTS categories (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NULL,
  type ENUM('expense','income') NOT NULL,
  name VARCHAR(50) NOT NULL,
  icon VARCHAR(100) NOT NULL,
  color VARCHAR(7) NULL,
  parent_id VARCHAR(36) NULL,
  sort_order INT NOT NULL DEFAULT 0,
  is_system TINYINT(1) NOT NULL DEFAULT 0,
  is_hidden TINYINT(1) NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_type (user_id, type),
  INDEX idx_parent (parent_id),
  INDEX idx_sort_order (sort_order),
  CONSTRAINT fk_category_parent FOREIGN KEY (parent_id) REFERENCES categories(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.4 创建账户表

```sql
-- 账户表
CREATE TABLE IF NOT EXISTS accounts (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  name VARCHAR(50) NOT NULL,
  type ENUM('cash','electronic','bank','credit','other') NOT NULL,
  icon VARCHAR(100) NULL,
  initial_balance DECIMAL(12,2) NOT NULL DEFAULT 0.00,
  current_balance DECIMAL(12,2) NOT NULL DEFAULT 0.00,
  is_default TINYINT(1) NOT NULL DEFAULT 0,
  is_active TINYINT(1) NOT NULL DEFAULT 1,
  sort_order INT NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  INDEX idx_is_default (is_default),
  INDEX idx_sort_order (sort_order),
  CONSTRAINT fk_account_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.5 创建账单表

```sql
-- 账单表
CREATE TABLE IF NOT EXISTS transactions (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  type ENUM('expense','income','transfer') NOT NULL,
  amount DECIMAL(12,2) NOT NULL,
  category_id VARCHAR(36) NULL,
  account_id VARCHAR(36) NOT NULL,
  to_account_id VARCHAR(36) NULL,
  transaction_date DATE NOT NULL,
  transaction_time TIME NULL,
  note VARCHAR(500) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  is_deleted TINYINT(1) NOT NULL DEFAULT 0,
  INDEX idx_user_id (user_id),
  INDEX idx_user_date (user_id, transaction_date),
  INDEX idx_user_category (user_id, category_id),
  INDEX idx_type (type),
  INDEX idx_is_deleted (is_deleted),
  CONSTRAINT fk_transaction_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_transaction_category FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE SET NULL,
  CONSTRAINT fk_transaction_account FOREIGN KEY (account_id) REFERENCES accounts(id) ON DELETE CASCADE,
  CONSTRAINT fk_transaction_to_account FOREIGN KEY (to_account_id) REFERENCES accounts(id) ON DELETE SET NULL,
  CONSTRAINT chk_amount_positive CHECK (amount > 0)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.6 创建预算表

```sql
-- 预算表
CREATE TABLE IF NOT EXISTS budgets (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  year_month VARCHAR(7) NOT NULL,
  category_id VARCHAR(36) NULL,
  budget_amount DECIMAL(12,2) NOT NULL,
  alert_threshold DECIMAL(5,2) NOT NULL DEFAULT 80.00,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_month (user_id, year_month),
  INDEX idx_user_category_month (user_id, category_id, year_month),
  UNIQUE KEY uk_user_category_month (user_id, category_id, year_month),
  CONSTRAINT fk_budget_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
  CONSTRAINT fk_budget_category FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.7 创建提醒表

```sql
-- 提醒表
CREATE TABLE IF NOT EXISTS reminders (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  reminder_time TIME NOT NULL,
  reminder_message VARCHAR(200) NULL,
  is_enabled TINYINT(1) NOT NULL DEFAULT 1,
  exclude_weekends TINYINT(1) NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  CONSTRAINT fk_reminder_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3.8 创建备份记录表

```sql
-- 备份记录表
CREATE TABLE IF NOT EXISTS backups (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL,
  backup_type ENUM('manual','auto') NOT NULL,
  file_url VARCHAR(500) NULL,
  file_size BIGINT NULL,
  status ENUM('success','failed') NOT NULL,
  error_message VARCHAR(500) NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_user_id (user_id),
  INDEX idx_created_at (created_at),
  CONSTRAINT fk_backup_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
