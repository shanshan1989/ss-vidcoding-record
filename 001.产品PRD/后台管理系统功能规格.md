# 随手记账 - 后台管理系统功能规格书

## 1. 概述

### 1.1 产品背景
为随手记账应用搭建独立的后台管理系统（Admin Management System），供运营人员查看系统注册用户、用户账单数据、收入支出等核心业务数据。

### 1.2 设计原则
- **只读优先**：管理员可查看所有数据，原则上不提供删除/修改功能
- **数据隔离**：严格区分不同用户的数据
- **性能优先**：支持分页、筛选、排序

### 1.3 约束条件
- **禁止修改现有数据库表结构**（现有 7 张业务表：`users`/`categories`/`accounts`/`transactions`/`budgets`/`reminders`/`backups`）
- **仅允许新增一张数据库表**：`admin_users`（管理员账户表），用于后台管理系统登录认证
- 沿用现有数据库，不修改任何现有表

---

## 2. 功能模块总览

| 模块 | 功能数量 | 说明 |
|------|---------|------|
| A. 仪表盘概览 | 4 | 系统核心指标总览 |
| B. 用户管理 | 5 | 注册用户列表及详情 |
| C. 账单管理 | 6 | 所有用户账单查询 |
| D. 收入支出分析 | 5 | 系统级收支统计 |
| E. 账户管理 | 3 | 用户账户查询 |
| F. 预算管理 | 2 | 用户预算查询 |

## 3. 功能详情

### 3.1 模块 A：仪表盘概览

#### A1. 系统核心指标卡片
- **指标项**：
  - 总注册用户数
  - 今日新增注册用户数
  - 本月总支出（系统所有用户合计）
  - 本月总收入（系统所有用户合计）
- **展示形式**：顶部 4 个统计卡片，等宽排列

#### A2. 近 30 天新增用户趋势图
- **图表类型**：折线图 / 柱状图
- **X 轴**：日期（最近 30 天）
- **Y 轴**：新增用户数
- **数据来源**：`users.created_at`

#### A3. 近 30 天系统收支趋势图
- **图表类型**：双折线图（收入+支出）
- **X 轴**：日期（最近 30 天）
- **Y 轴**：金额
- **数据来源**：`transactions`

#### A4. 最近新注册用户列表
- **展示列**：用户名、昵称、注册时间
- **行数限制**：最近 10 条
- **排序**：按注册时间倒序

---

### 3.2 模块 B：用户管理

#### B1. 用户列表页
- **页面路径**：`/admin/users`
- **筛选条件**：
  - 用户名（模糊搜索）
  - 注册日期范围（开始日期 ~ 结束日期）
  - 账户状态（全部 / 正常 / 已禁用）
- **列表字段**：
  | 字段 | 说明 |
  |------|------|
  | 用户ID | `id` |
  | 用户名 | `username` |
  | 昵称 | `nickname` |
  | 注册时间 | `created_at` |
  | 最后登录时间 | `last_login_at` |
  | 账户状态 | `is_active`（1=正常，0=已禁用） |
- **排序**：支持按注册时间、最后登录时间排序（默认注册时间倒序）
- **分页**：每页 20 条，支持跳转页码

#### B2. 用户详情页
- **页面路径**：`/admin/users/:id`
- **信息区块**：
  - **基本信息**：用户ID、用户名、昵称、头像、个性签名、币种、注册时间、最后登录时间
  - **账户概况**：账户数量、当前总余额
  - **本月统计**：本月收入合计、本月支出合计
- **数据来源**：`users`、`accounts`、`transactions`

#### B3. 用户账单快速入口
- **位置**：用户详情页
- **功能**：点击跳转至「账单管理」页，并自动填充该用户ID为筛选条件

#### B4. 用户账户快速入口
- **位置**：用户详情页
- **功能**：点击跳转至「账户管理」页，并自动填充该用户ID为筛选条件

#### B5. 用户删除（软删除）
- **操作**：将用户的 `is_active` 设为 0
- **约束**：仅禁用登录，不删除任何业务数据
- **注意**：此操作修改用户表，但属于管理需要，标记为例外

---

### 3.3 模块 C：账单管理

#### C1. 账单列表页
- **页面路径**：`/admin/transactions`
- **筛选条件**：
  | 条件 | 类型 | 说明 |
  |------|------|------|
  | 用户 | 下拉选择 / 用户ID搜索 | 支持模糊搜索用户名 |
  | 账单类型 | 单选 | 全部 / 支出 / 收入 / 转账 |
  | 分类 | 下拉选择 | 仅支出/收入可选，分类列表需加载系统+用户自定义 |
  | 日期范围 | 日期区间 | 开始日期 ~ 结束日期，默认本月 |
- **列表字段**：
  | 字段 | 说明 |
  |------|------|
  | 账单ID | `id` |
  | 用户 | 用户名（可点击跳转用户详情） |
  | 类型 | expense=支出 / income=收入 / transfer=转账 |
  | 金额 | `amount`，收入显示绿色，支出显示红色，转账显示蓝色 |
  | 分类 | `category_id` 对应分类名（转账显示"—"） |
  | 账户 | 支出/收入显示账户名；转账显示"账户A → 账户B" |
  | 日期 | `transaction_date`，格式 YYYY-MM-DD |
  | 时间 | `transaction_time`（如有） |
  | 备注 | `note`（超出部分省略） |
- **排序**：支持按日期、金额排序（默认日期倒序）
- **分页**：每页 50 条

#### C2. 账单详情页
- **页面路径**：`/admin/transactions/:id`
- **展示信息**：账单全部字段 + 关联的用户名、账户名、分类名

#### C3. 按用户汇总账单
- **功能**：选择用户后，展示该用户所有账单按月汇总的表格
- **汇总字段**：月份、总收入、总支出、净收支（收入-支出）、账单数量

#### C4. 批量导出
- **功能**：将当前筛选条件下的账单导出为 CSV 文件
- **导出字段**：同列表字段

#### C5. 支出分类分布图
- **页面**：账单管理页内嵌
- **图表类型**：饼图 / 环形图
- **数据**：当前筛选条件下的支出金额按分类分布
- **交互**：点击分类区块可跳转至该分类的账单列表

#### C6. 账单搜索
- **功能**：支持按备注内容关键词搜索账单

---

### 3.4 模块 D：收入支出分析

#### D1. 全局收支概览页
- **页面路径**：`/admin/statistics/overview`
- **指标卡片**：
  - 累计总收入
  - 累计总支出
  - 累计净利润（收入-支出）
  - 累计账单数

#### D2. 按月收支趋势图
- **图表类型**：双柱状图（收入 vs 支出）
- **X 轴**：月份
- **Y 轴**：金额
- **时间范围**：最近 12 个月
- **交互**：点击某月柱状图跳转至该月账单列表

#### D3. 支出分类占比分析
- **页面路径**：`/admin/statistics/expense-breakdown`
- **筛选条件**：月份下拉选择
- **图表**：饼图 + 排行榜表格
- **排行榜字段**：分类名、图标、颜色、总金额、占比%

#### D4. 收入分类占比分析
- **页面路径**：`/admin/statistics/income-breakdown`
- **结构同 D3**，但数据为收入分类

#### D5. 用户消费排行榜
- **页面路径**：`/admin/statistics/user-ranking`
- **筛选条件**：月份下拉选择
- **排行榜字段**：排名、用户名、昵称、本月支出总额、账单数量、平均单笔金额

---

### 3.5 模块 E：账户管理

#### E1. 账户列表页
- **页面路径**：`/admin/accounts`
- **筛选条件**：
  | 条件 | 说明 |
  |------|------|
  | 用户 | 下拉选择 / 用户名搜索 |
  | 账户类型 | 全部 / 现金 / 电子钱包 / 银行卡 / 信用卡 / 其他 |
  | 账户状态 | 全部 / 正常 / 已删除 |
- **列表字段**：
  | 字段 | 说明 |
  |------|------|
  | 账户ID | `id` |
  | 用户 | 用户名（可点击跳转用户详情） |
  | 账户名称 | `name` |
  | 类型 | `type`（中文显示） |
  | 期初余额 | `initial_balance` |
  | 当前余额 | `current_balance` |
  | 是否默认 | `is_default` |
  | 状态 | `is_active` |
  | 创建时间 | `created_at` |
- **排序**：支持按用户、当前余额、创建时间排序
- **分页**：每页 20 条

#### E2. 账户详情页
- **页面路径**：`/admin/accounts/:id`
- **展示信息**：
  - 账户全部字段
  - 关联用户名
  - 该账户最近 10 笔账单

#### E3. 用户账户余额汇总
- **功能**：选择用户后，展示该用户所有账户的余额汇总（现金+电子+银行+信用+其他=总资产）

---

### 3.6 模块 F：预算管理

#### F1. 预算列表页
- **页面路径**：`/admin/budgets`
- **筛选条件**：
  | 条件 | 说明 |
  |------|------|
  | 用户 | 下拉选择 |
  | 月份 | YYYY-MM 下拉选择 |
- **列表字段**：
  | 字段 | 说明 |
  |------|------|
  | 预算ID | `id` |
  | 用户 | 用户名 |
  | 月份 | `year_month` |
  | 预算类型 | 总预算 / 分类预算（`category_id` 为空=总预算） |
  | 分类 | 若为分类预算，显示分类名；否则显示"全局" |
  | 预算金额 | `budget_amount` |
  | 警戒阈值 | `alert_threshold`（百分比） |
  | 创建时间 | `created_at` |

#### F2. 预算使用情况
- **功能**：展示预算的实际使用情况
- **计算方式**：`已使用金额 / 预算金额`
- **展示形式**：进度条 + 百分比

## 4. 管理员认证

### 4.1 登录页
- **页面路径**：`/admin/login`
- **字段**：管理员用户名、密码
- **说明**：后台管理员独立于业务用户，使用专用的 `admin_users` 表存储账号

### 4.2 管理员表（`admin_users`）

> **仅此一张新增表**，用于存放后台管理系统管理员账号。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | VARCHAR(36) | PRIMARY KEY | UUID |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE | 管理员登录用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | bcrypt 加密后的密码 |
| `nickname` | VARCHAR(50) | NULL | 显示名称 |
| `is_active` | TINYINT(1) | NOT NULL, DEFAULT 1 | 账户状态（1=启用，0=禁用） |
| `created_at` | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | NOT NULL, ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

- **Indexes**：`idx_username`

### 4.3 认证方式
- 采用独立的管理员账户体系，与业务用户 `users` 表完全分离
- 登录时验证 `username` + `password_hash`（bcrypt 比对）
- 使用 Session/Cookie 机制维持登录状态

### 4.4 权限控制
- 所有 `/admin/*` 路由需经过管理员身份验证
- 未登录访问任何管理页面统一跳转至登录页
- 登录状态由服务端 session 管理，不依赖 localStorage

---

## 5. 技术约束

| 约束项 | 说明 |
|--------|------|
| 数据库 | 禁止修改现有 7 张业务表结构；仅允许新增 `admin_users` 表 |
| API 认证 | 管理员使用独立 session 机制，与业务用户 API 分离 |
| 前端渲染 | 可使用服务端渲染（SSR）或纯前端方案 |
| 数据导出 | 支持 CSV 格式导出 |
| 管理员表 | `admin_users` 为唯一新增表，用于管理员登录认证 |
