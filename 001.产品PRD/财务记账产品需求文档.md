## 1. 产品背景与目标

### 1.1 背景

用户每天都有消费行为，但大多数人对钱花在哪里、是否超支缺乏清晰认知。现有的记账工具要么功能臃肿（广告、理财、社区），要么过于简单（无法分类、无统计），导致用户难以坚持。

本产品希望做一款**轻量、专注、易坚持**的个人记账工具，帮助用户养成每日记账习惯，并通过数据反馈实现消费自律。

### 1.2 产品目标

| 目标层级 | 描述 |
|---------|------|
| 核心目标 | 让用户每天用最少的操作完成记账 |
| 延伸目标 | 帮助用户看清消费结构，控制不必要支出 |
| 商业目标 | 先验证用户粘性，后续可考虑会员高级报表/数据导出等增值服务 |

### 1.3 成功指标（V1.0）

- 日活跃用户中，单日记账 ≥1 次的比例达到 60%
- 新用户 7 日留存率 ≥40%
- 用户平均单次记账操作步骤 ≤3 步
- 月度活跃用户中，查看报表的比例 ≥50%

## 2. 用户画像

### 2.1 目标用户

**主要用户：年轻上班族 / 学生**

- 年龄：20-35 岁
- 特征：有手机记账意识，但难以坚持
- 痛点：
  - 记账步骤繁琐，经常忘记
  - 不知道钱花在哪里
  - 月初没预算，月底吃土
  - 担心记账 App 广告多、隐私差

### 2.2 用户故事

> 作为上班族小王，我希望能 3 秒内记一笔午餐开销，这样我才能在吃饭间隙坚持记账。

> 作为学生小李，我希望看到本月在餐饮、娱乐上花了多少钱，这样我能控制自己的非必要消费。

> 作为用户小张，我希望设置每月预算并在接近上限时收到提醒，这样我不会再超支。

## 3. 产品定位

- **Slogan**：「每天三秒，钱花得明白」
- **定位**：极简个人记账工具
- **差异化**：
  - 打开即记，无需进入多级页面
  - 无广告、无理财推荐、无社区
  - 默认分类清晰，支持自定义
  - 数据本地优先，隐私可控

## 5. 核心功能详细说明

### 5.1 用户认证

**需求描述**：

用户需要注册账号并登录后才能使用记账功能，实现多用户数据隔离。

**功能细节**：

- **注册**：
  - 支持用户名 + 密码注册
  - 用户名至少 2 个字符
  - 密码至少 6 个字符
  - 用户名不可重复
  - 注册成功后自动跳转登录页面

- **登录**：
  - 支持用户名 + 密码登录
  - 登录成功后加载用户专属数据
  - 记住登录状态（基于 localStorage）

- **退出**：
  - 「我的」页面显示当前用户名
  - 提供退出登录按钮
  - 退出后跳转登录页面

- **数据隔离**：
  - 每个用户的数据独立存储
  - 数据以用户 ID 为 key 区分
  - 同一浏览器可支持多账号切换

### 5.2 快速记账

**需求描述**：

用户进入 App 首页后，应能在最显眼的位置完成记账。

**功能细节**：

- 首页顶部显示「本月结余」和「今日支出」
- 首页中部为数字键盘，底部为分类选择栏
- 输入金额 → 选择分类 → 点击确认，完成记账
- 支持添加备注（可选，默认不展开）
- 支持修改账单时间（默认当前时间，可选）
- 支持选择支出账户（默认上次使用账户）

**默认支出分类**：

| 一级分类 | 示例二级分类 |
|---------|------------|
| 餐饮 | 早餐、午餐、晚餐、零食、饮料 |
| 交通 | 公交、地铁、打车、加油、停车 |
| 购物 | 服饰、日用、数码、美妆 |
| 居住 | 房租、水电、物业、宽带 |
| 娱乐 | 电影、游戏、旅行、会员 |
| 医疗 | 药品、挂号、体检 |
| 学习 | 书籍、课程、考试 |
| 其他 | 礼物、捐赠、其他 |

**默认收入分类**：

- 工资、奖金、兼职、理财收益、红包、退款、其他

### 5.3 账单明细

**需求描述**：

用户可以按时间查看所有账单，并支持编辑和删除。

**功能细节**：

- 列表按日期分组，显示当日总收支
- 每条账单显示：分类图标、名称、备注、金额、账户
- 左滑删除，点击进入编辑
- 顶部支持月份切换
- 支持关键词搜索（分类名、备注）

### 5.4 统计报表

**需求描述**：

通过可视化图表帮助用户理解消费结构。

**功能细节**：

- **月度总览**：本月收入、支出、结余
- **支出分类饼图**：按一级分类汇总，点击可下钻二级分类
- **支出趋势图**：近 6 个月支出折线图
- **收支对比图**：本月每日收入/支出柱状图
- **排行榜**：本月支出最多的分类 TOP5

### 5.5 预算管理

**需求描述**：

用户可以设置月度预算，系统实时反馈预算使用情况。

**功能细节**：

- 支持设置月度总预算
- 支持为单个分类设置预算（可选）
- 首页显示预算进度条
- 当支出达到预算的 80% 时，显示黄色预警
- 当支出达到或超过预算时，显示红色超支提示
- 支持每月自动重置预算

### 5.6 账户管理

**需求描述**：

管理用户的资金来源，支持多账户记账。

**功能细节**：

- 默认账户：现金、微信钱包、支付宝、银行卡、信用卡
- 每个账户显示当前余额
- 支持账户间转账（不记收入/支出，只调整余额）
- 记支出时自动扣减对应账户余额
- 记收入时自动增加对应账户余额

### 5.7 分类管理

**需求描述**：

用户可以自定义分类，满足个性化需求。

**功能细节**：

- 支持新增、编辑、删除自定义分类
- 支持为分类选择图标和颜色
- 默认分类不可删除，但可隐藏
- 支持调整分类排序

### 5.8 数据安全与备份

**需求描述**：

确保用户数据不丢失，同时保护隐私。

**功能细节**：

- 数据默认本地存储
- 支持 iCloud/Google Drive/本地文件 备份
- 支持从备份恢复
- 支持导出 CSV/Excel 文件
- 可选开启云同步（V2.0 规划）

### 5.9 记账提醒

**需求描述**：

通过温和提醒帮助用户养成记账习惯。

**功能细节**：

- 支持设置每日提醒时间（默认 21:00）
- 提醒文案温和，例如：「今天记账了吗？」
- 支持关闭提醒
- 支持周末不提醒（可选）

## 6. 用户流程

### 6.1 首次使用流程

```
打开 App → 注册页面 → 输入用户名/密码 → 注册成功 → 登录 → 进入首页
```

### 6.2 日常使用流程

```
打开 App → 登录页面 → 输入用户名/密码 → 登录成功 → 进入首页
```

### 6.3 日常记账流程

```
打开 App（首页即记账页面）
  ↓
输入金额
  ↓
选择分类（默认上次分类或智能推荐）
  ↓
点击「确认」（可选：填写备注、修改时间、选择账户）
  ↓
记账成功，显示轻提示，首页数据实时更新
```

### 6.4 查看报表流程

```
底部导航切换到「统计」
  ↓
默认显示本月总览
  ↓
切换顶部 Tab：分类 / 趋势 / 对比
  ↓
点击图表下钻查看明细
```

| 字段 | 类型 | 说明 |

| type | Enum | 支出 / 收入 |
| icon | String | 图标名称 |

| parentId | String | 父分类 ID（空为一级分类）|

| 字段 | 类型 | 说明 |
|-----|------|------|

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | String | 唯一标识 |

| amount | Decimal | 预算金额 |

## 8. 界面原型说明

### 8.1 首页（记账页）

- **顶部**：本月结余卡片（大字号显示结余，小字显示本月收入/支出）
- **中部**：预算进度条（如有设置）
- **下部**：分类快捷选择栏（横向滑动，8 个常用分类 + 「更多」）
- **底部**：数字键盘 + 确认按钮
- **操作**：输入金额后，选择分类，点击确认完成记账

### 8.2 明细页

- **顶部**：月份选择器 + 搜索图标
- **中部**：当月总收入/支出/结余
- **下部**：按日期分组的账单列表
- **空状态**：显示「本月还没有记账哦，快去记一笔吧」

### 8.3 统计页

- **顶部**：月份选择器
- **Tab 切换**：总览 / 分类 / 趋势 / 对比
- **总览**：大数字 + 收支对比卡片
- **分类**：饼图 + 分类列表
- **趋势**：折线图 / 柱状图

### 8.4 我的页

- 账户管理
- 分类管理
- 预算设置
- 数据备份与导出
- 记账提醒
- 主题设置
- 关于与反馈

## 9. 非功能需求

### 9.1 性能

- 首页冷启动时间 ≤2 秒
- 记账操作响应时间 ≤200ms
- 统计图表加载时间 ≤1 秒

### 9.2 兼容性

- 支持 iOS 14+ / Android 8+
- 适配手机竖屏，暂不支持平板和横屏

### 9.3 安全与隐私

- 用户数据默认本地存储
- 不强制要求手机号/邮箱注册
- 不上传用户消费数据（除非用户主动开启云同步）

### 9.4 可访问性

- 支持系统字体大小设置
- 图标 + 文字双标识
- 关键操作有触觉反馈

## 12. 风险与假设

### 12.1 风险

| 风险 | 影响 | 应对策略 |
|-----|------|---------|
| 用户记账坚持度低 | 高 | 强化提醒、成就体系、极简流程 |
| 同类竞品竞争激烈 | 中 | 差异化定位：极简、无广告、隐私优先 |
| 数据丢失导致用户流失 | 高 | 强调备份功能，提供自动备份选项 |

### 12.2 关键假设

- 用户愿意为「简洁无广告」的记账体验付费或接受基础功能免费
- 用户主要使用少数几个固定分类记账
- 移动端是个人记账的主要场景
